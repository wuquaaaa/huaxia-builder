# 猫国建造者（Cat Country Builder）设计文档

> 一款以猫为主题的放置/增量经营游戏（Idle / Incremental Game）。玩家从一只孤独的猫开始，逐步发展出一个文明：采集资源、修建建筑、研究科技、管理人口、应对四季更替，最终走向贸易、宗教乃至太空殖民。
>
> 设计灵感来源于经典网页游戏《Kittens Game》（中文社区译名即"猫国建造者"），本文档在其框架基础上整理出可实现的系统设计。

---

## 1. 设计概述

### 1.1 核心定位
- **类型**：放置 / 增量 / 文明经营（Idle / Incremental / Civilization Management）
- **平台**：网页（HTML5 + JavaScript），可移植到移动端
- **视角**：纯数值面板式 UI，无即时动画战斗，强调"看着数字增长"的满足感
- **节奏**：前期手动点击采集，中期建筑自动化产出，后期靠科技与转生（Reset）系统循环成长

### 1.2 核心循环（Core Loop）
```
采集资源 → 建造建筑 → 解锁科技 → 增加人口 → 分配工种 →
提升产能 → 解锁更高级资源/建筑 → 应对瓶颈（季节/储量上限/信仰）→
转生重置获得永久加成 → 更快地循环
```

### 1.3 设计目标
1. **正反馈密集**：每一次点击、每一次升级都有可见数值回报。
2. **决策有深度**：资源相互转化，储量有上限，工种分配需要权衡。
3. **长线留存**：通过转生（Paragon/虔诚）系统提供"重开更强"的长期目标。
4. **离线友好**：关闭游戏后资源仍按比例累积。

## 2. 资源系统（Resources）

资源是游戏的核心货币。分为基础资源、加工资源、稀有资源与特殊资源四类。每种资源都有**产量（per tick）**与**储量上限（cap）**两个关键属性。

### 2.1 基础资源
| 资源 | 来源 | 说明 |
|------|------|------|
| 猫薄荷（Catnip） | 采集 / 农田 | 最基础食物，喂养人口、冬季消耗加剧 |
| 木材（Wood） | 猫薄荷转化 / 伐木工 | 建筑核心材料 |
| 矿物（Minerals） | 矿工 | 建筑与加工材料 |
| 毛皮（Furs） | 狩猎副产 | 贸易与高级建筑材料 |

### 2.2 加工资源
| 资源 | 来源 | 用途 |
|------|------|------|
| 铁（Iron） | 冶炼厂消耗矿物 | 进阶建筑、工具 |
| 煤（Coal） | 深井矿 / 魔法 | 钢铁与能源 |
| 钢（Steel） | 铁 + 煤 | 高级建筑、太空 |
| 木板（Slab） | 矿物加工 | 大型建筑地基 |
| 钛（Titanium） | 海港贸易 / 加速器 | 高级科技建筑 |

### 2.3 稀有 / 特殊资源
| 资源 | 说明 |
|------|------|
| 黄金（Gold） | 贸易、宗教、高级转化的硬通货 |
| 象牙（Ivory） | 与"猛犸象人"贸易获得 |
| 香料（Spice） | 贸易获得，提升幸福度 |
| 文化（Culture） | 圆形剧场 / 图书馆产出，影响科技与外交 |
| 科学（Science） | 学者产出，用于解锁科技 |
| 信仰（Faith） | 神庙 / 牧师产出，宗教系统核心 |
| 星陨石（Unobtainium） | 月球矿场，后期顶级材料 |

### 2.4 储量上限机制
- 每种资源都有 cap，超出部分被浪费。
- 提升 cap 是中期核心目标：仓库（Warehouse）、谷仓（Barn）、港口（Harbour）等专门用于扩容。
- 设计意图：迫使玩家在"建仓扩容"与"消耗产出"之间做权衡，避免无脑囤积。

## 3. 建筑系统（Buildings）

建筑是产能的来源，绝大多数建筑成本随数量呈**指数增长**（典型系数 1.12～1.15），形成天然的成长曲线。

### 3.1 生产类
- **田地（Field）**：产猫薄荷，受季节系数影响（春+，冬-）。
- **小屋（Hut）**：增加人口上限（每座 +2 居民），消耗猫薄荷。
- **小屋升级 → 木屋 / 别墅**：提供更高人口上限与少量幸福度。

### 3.2 加工类
- **伐木场（Lumber Mill）**：提升木材产量。
- **冶炼厂（Smelter）**：消耗矿物与木材产出铁（消耗型建筑，可开关）。
- **磁化器 / 加速器**：后期提升整体产能或转化稀有资源。

### 3.3 存储类
- **仓库 / 谷仓 / 港口**：提升各类资源 cap。
- 设计上需要玩家持续投入，否则产能会被 cap 卡死。

### 3.4 知识与文化类
- **图书馆（Library）**：提升科学 cap 与产量。
- **学院（Academy）**：进一步提升科研效率。
- **圆形剧场（Amphitheatre）**：产出文化、降低不满。
- **神庙（Temple）**：产出信仰与文化，宗教系统入口。

### 3.5 高级 / 后期建筑
- **太阳能上层建筑、磁圈、计算核心**：提升能源与全局加成。
- **航天中心、月球基地、轨道阵列**：解锁太空玩法。

> **设计要点**：建筑分"被动产出"与"开关消耗型"两类。消耗型建筑（如冶炼厂）让玩家主动管理能源与原料平衡，增加策略性。

## 4. 人口与工种系统（Kittens & Jobs）

### 4.1 人口增长
- 人口（猫）随时间在"人口上限"内自然增长，前提是猫薄荷储备为正。
- 食物不足时会触发**饥荒**，猫会饿死——这是核心的负反馈约束。

### 4.2 工种分配
玩家将每只猫分配到一种职业，决定其产出：

| 工种 | 产出 | 解锁条件 |
|------|------|----------|
| 农夫（Farmer） | 猫薄荷 | 初始 |
| 伐木工（Woodcutter） | 木材 | 初始 |
| 学者（Scholar） | 科学 | 解锁科技 |
| 猎人（Hunter） | 毛皮 / 象牙 | 解锁狩猎 |
| 矿工（Miner） | 矿物 | 解锁采矿 |
| 地质学家（Geologist） | 矿物 / 稀有 | 进阶 |
| 牧师（Priest） | 信仰 / 文化 | 宗教 |

### 4.3 技能成长
- 每只猫从事某职业越久，**熟练度（skill）**越高，产量加成提升。
- 设计意图：让玩家不愿轻易调换工种，形成"养成"黏性；同时"领袖"系统可指定一只猫为领袖获得全局加成。

### 4.4 幸福度（Happiness）
- 受人口数量（多则降低）、奢侈品消费（毛皮、象牙、香料等提升）、文化建筑影响。
- 幸福度直接乘算到所有产出，是后期重要的优化维度。

---

## 5. 科技与文化系统（Science & Culture）

### 5.1 科技树（Tech Tree）
- 用**科学**解锁，呈树状依赖。
- 关键节点示例：农业 → 畜牧 → 采矿 → 冶金 → 工程学 → 数学 → 物理 → 化学 → 电气 → 计算机 → 太空科学。
- 每解锁一项科技，会开放新建筑、新工种、新资源或全局加成。

### 5.2 工程升级（Workshop / Crafting）
- 工坊允许将基础资源**手工合成**为加工资源（如木材→木板，铁+煤→钢）。
- 工坊升级（Upgrades）提升合成效率、降低损耗、提升 cap。

### 5.3 文化与外交
- 文化值影响外交选项与部分科技解锁。
- 后期"政体（Government）"可选择共和、君主、寡头等，提供不同的全局倾向加成。

---

## 6. 季节与历法（Seasons & Calendar）

### 6.1 四季循环
游戏内时间以"天 → 季 → 年"推进，一年四季：

| 季节 | 影响 |
|------|------|
| 春 | 猫薄荷产量 +50%，发展黄金期 |
| 夏 | 正常产量 |
| 秋 | 正常，准备过冬 |
| 冬 | 猫薄荷产量大幅下降（-75%），食物消耗压力陡增 |

### 6.2 天气事件
- 随机出现"寒冬""暖春"等天气，进一步上下浮动产量。
- 设计意图：制造资源储备规划压力，玩家需在丰季囤粮以渡过冬季。

### 6.3 日历升级
- 解锁"历法"科技后可预知季节，进而规划农田与仓储节奏。

---

## 7. 贸易与外交（Trade & Diplomacy）

### 7.1 种族（Races）
通过探索解锁可贸易的智慧种族，各有偏好与特产：

| 种族 | 特产 | 偏好 |
|------|------|------|
| 狮族（Lizards） | 矿物 / 木板 | 猫薄荷 |
| 猛犸象人（Sharks/Nagas 等） | 象牙 / 钛 | 铁 / 香料 |
| 灰族（Griffins） | 铁 | 木材 |
| 龙族（Dragons） | 钛 / 星陨石 | 黄金（高级） |

### 7.2 贸易机制
- 每次贸易消耗黄金 + 特定资源，按概率换回对方特产。
- 关系（Standing）影响成功率与产出，可通过外交礼物提升。
- 设计意图：贸易是获取**钛、象牙**等无法本地生产资源的主要途径，构成中期资源闭环的关键。

---

## 8. 宗教系统（Religion）

### 8.1 信仰积累
- 牧师与神庙产出**信仰**。
- 信仰可"献祭"转化为**虔诚（Epiphany）**，进而升华为永久加成。

### 8.2 转化生命树（Transcendence）
- 高级宗教升级提供全局产能、储量与外交的乘算加成。
- 是中后期"质变"的重要来源。

### 8.3 太阳信仰 / 异端信仰
- 提供两条不同的宗教升级路线，分别侧重生产与储量，鼓励 build 多样化。

---

## 9. 转生与元进度（Reset & Meta-progression）

放置游戏的灵魂在于"重开更强"。

### 9.1 重置（Reset）
- 玩家可主动重置游戏，将累计成就转化为**纸星（Karma）**与**领悟点（Paragon）**。
- Paragon 提供永久的产量与储量百分比加成，重置后保留。

### 9.2 设计意图
- 第一周目可能需要数十小时，重置后凭借 Paragon 加成可在数小时内重达原进度并突破更远。
- 形成"螺旋上升"的长期目标，是留存的核心。

---

## 10. 太空殖民（Space / 后期内容）

- 解锁航天科技后开放太空面板：建造卫星、登陆月球、开采星陨石。
- 太空建筑产出顶级资源，并解锁全新科技分支（如反物质、奇点）。
- 作为终局内容（Endgame），为硬核玩家提供长达数百小时的追求目标。

- **颜色编码**：资源接近上限变黄、净产为负变红。

## 12. 技术实现建议

### 12.1 架构
- **状态层**：单一全局 game state 对象（资源、建筑、科技、人口等），易于存档序列化。
- **tick 循环**：固定时间步（如每秒 5 tick），每 tick 计算产出、消耗、增长。
- **离线结算**：记录退出时间戳，重进时按差值批量结算（设上限防滥用）。

### 12.2 数据驱动
- 建筑、科技、资源全部用配置表（JSON）定义，逻辑与数据分离，方便平衡调整与扩展。

### 12.3 存档
- localStorage 自动存档 + 手动导出/导入（Base64 字符串），无需服务器。

### 12.4 数值平衡
- 建筑成本指数系数控制成长曲线（建议 1.12～1.15）。
- 用电子表格预先建模前 50 次购买的成本与产出，确保不会过快或过慢。

| MVP | 猫薄荷采集 + 田地 + 小屋 + 人口 + tick 循环 + 存档 |

## 14. 附录：核心公式参考

- **建筑成本**：`cost = baseCost × ratio^(已建数量)`，ratio ≈ 1.12
- **资源产量**：`产出 = 基础产量 × 数量 × (1 + 工种熟练度) × 季节系数 × 幸福度 × 全局加成`
- **人口增长**：`Δpop = growthRate × (当前人口) × (1 - 当前/上限)`，食物为负时转为死亡
- **离线收益**：`offline = min(实际离线时长, 上限) × 每秒净产`
